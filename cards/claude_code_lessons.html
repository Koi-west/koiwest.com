⚡ Prompt Caching Is Everything
LAY OUT FOR CACHING
Static first, dynamic last. 顺序决定命中率： System Prompt & Tools → Claude.MD → Session Context → Conversation
USE MESSAGES FOR UPDATES
信息变更时不要改 System Prompt，用 <system-reminder> 标签插入到下一条 message 中
DON'T CHANGE MODELS
中途切换模型会重建缓存。100k tokens 后用 Haiku 比重新缓存更贵。用 Subagent Handoff 模式
NEVER ADD/REMOVE TOOLS
Tools 是缓存前缀的一部分。Plan Mode 用 EnterPlanMode/ExitPlanMode 作为工具本身，而非切换工具集
DEFER LOADING
数十个 MCP 工具时用 stub + defer_loading: true，模型需要时通过 ToolSearch 发现
CACHE-SAFE FORKING
Compaction 时用父对话完全相同的 System Prompt + Tools，只追加 compact prompt 作为新 message

▣ Making Skills — Core Tips
DON'T STATE THE OBVIOUS
只写 Claude 不知道的知识。Frontend Design Skill 通过迭代客户反馈，避开 Inter 字体和紫色渐变
BUILD A GOTCHAS SECTION
最高信号内容。随时间累积常见失败点，持续更新 Skill
FILE SYSTEM = PROGRESSIVE DISCLOSURE
Skill 是文件夹不是 markdown。用 references/api.md 放详细签名，assets/ 放模板
AVOID RAILROADING
给信息但不锁死。让 Claude 有灵活性适应不同情境
DESCRIPTION IS FOR TRIGGERING
Description 不是摘要，是「何时触发此 Skill」的信号
STORE SCRIPTS & GENERATE CODE
给 Claude 函数库让它做组合而非 boilerplate。Data Science Skill 给 fetch 函数让 Claude 写分析脚本
ON DEMAND HOOKS
/careful блокирует rm -rf、DROP TABLE。只在 touch prod 时启用
MONITOR CACHE HIT RATE
像监控 Uptime 一样监控缓存命中率。Anthropic 对 cache break declare SEVs
